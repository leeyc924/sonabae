- 문서 버전: v1.0
- 작성일: 2026-05-11
- 적용 제품: 소중한 나의 배드민턴 일지, “소나배”

---

## 1. 디자인 방향

## 1.1 디자인 콘셉트

**Sporty Calm Dashboard**

소나배는 배드민턴 동호인을 위한 운동 기록 앱이다. 따라서 디자인은 다음 느낌을 가져야 한다.

- 운동 앱답게 경쾌하다.
- 기록 앱답게 명확하다.
- 승급 목표를 자극하되 과하게 경쟁적이지 않다.
- 경기 직후 빠르게 기록할 수 있도록 단순하다.
- 통계 화면은 숫자 중심이지만 차갑지 않다.

## 1.2 핵심 디자인 문장

> “운동 후 30초 안에 기록하고, 내 승률과 급수를 한눈에 확인한다.”

---

## 2. 2026+ 디자인 레퍼런스 반영 방향

## 2.1 Liquid Glass / Soft Depth

2026년 모바일 UI 트렌드에서는 투명감, 깊이감, 레이어감을 활용한 Liquid Glass 계열 디자인이 강조된다. 소나배에서는 이를 과하게 적용하지 않고 다음처럼 제한적으로 사용한다.

### 적용 위치

- 홈 상단 요약 카드
- 현재 급수 카드
- 승률 하이라이트 카드
- 플로팅 기록 버튼
- 모달 배경

### 주의사항

- 본문 텍스트 위에는 강한 투명 효과를 쓰지 않는다.
- 통계 숫자의 가독성을 해치지 않는다.
- 배경이 복잡한 경우 카드 투명도를 낮추거나 단색 카드로 전환한다.

---

## 2.2 Material 3 Expressive / 감정 있는 데이터 UI

Material 3 Expressive 방향은 색상, 타이포그래피, 모션, 컴포넌트를 더 생동감 있게 사용하도록 제안한다. 소나배에서는 승률과 급수 정보를 단순 표가 아니라 “운동 성과 카드”처럼 보여준다.

### 적용 방향

- 승률이 좋은 항목은 긍정적인 강조 표현을 사용한다.
- 최근 기록이 없는 경우 부드러운 빈 상태 메시지를 보여준다.
- 급수와 목표 급수는 배지 형태로 표현한다.
- 숫자는 크고 명확하게, 보조 설명은 작게 표시한다.

---

## 2.3 Motion-driven Interface

모션은 장식이 아니라 사용자의 이해를 돕는 도구로 사용한다.

### 적용 모션

- 기록 저장 완료 시 짧은 성공 피드백
- 승률 카드 진입 시 숫자 카운트업
- 탭 전환 시 부드러운 슬라이드
- 경기 추가 시 리스트에 자연스럽게 삽입
- 급수 배지 변경 시 작은 강조 애니메이션

### 금지

- 통계 확인을 방해하는 긴 애니메이션
- 매번 반복되는 과한 축하 효과
- 화면 전환을 느리게 만드는 모션

---

## 2.4 Predictive / Personalized UI

사용자가 반복적으로 선택하는 모임, 사람, 경기 방식을 기반으로 다음 행동을 쉽게 만든다.

### 적용 예시

- 최근 선택한 모임을 기록 화면 기본값으로 설정
- 최근 함께 경기한 사람을 상단에 노출
- 자주 쓰는 경기 유형을 우선 선택
- 홈에서 “지난번 모임 이어서 기록” 버튼 제공

---

## 2.5 접근성 우선

소나배는 통계와 숫자가 많은 앱이므로 접근성이 매우 중요하다.

### 원칙

- 색상만으로 승/패를 구분하지 않는다.
- 승률 숫자에는 텍스트 라벨을 함께 제공한다.
- 터치 영역은 충분히 크게 만든다.
- React Native 접근성 속성을 적용한다.
- 카드형 UI에서도 텍스트 대비를 충분히 확보한다.

## 3.1 앱명

## 3.2 브랜드 키워드

- 소중함
- 기록
- 승률
- 성장
- 승급
- 셔틀콕
- 코트
- 동호회
- 꾸준함

## 3.3 톤 앤 매너

- 친근하지만 가볍지 않다.
- 운동 의욕을 높인다.
- 사용자를 평가하지 않고 성장 중심으로 말한다.
- 실패보다 다음 경기를 준비하게 만든다.

### 좋은 문구 예시

- “오늘 경기를 기록해볼까요?”
- “최근 승률이 좋아지고 있어요.”
- “이 모임에서는 꾸준히 성장 중이에요.”
- “다음 급수까지 한 걸음 더!”

### 피해야 할 문구

- “실력이 부족합니다.”
- “패배가 너무 많습니다.”
- “당신은 약합니다.”
- “승급 가능성이 낮습니다.”

## 8. 화면 디자인 가이드

## 8.1 홈 화면

### 목적

오늘의 기록 시작과 핵심 성과 확인.

### 구성 순서

1. 인사말
2. 현재 급수 / 목표 급수 카드
3. 전체 승률 카드
4. 최근 30일 승률 카드
5. 최근 운동일지
6. 플로팅 기록 버튼

### 예시 구조

```text
안녕하세요, 오늘도 한 게임 기록해볼까요?

[현재 C조 | 목표 B조]

[전체 승률 64.2%]
[최근 30일 승률 70.1%]

최근 기록
- 5월 10일 한강클럽 3승 1패
- 5월 08일 수요모임 2승 2패

[+ 기록하기]
```

---

## 8.2 기록하기 화면

### 목적

경기 후 빠르게 기록.

### 구성

1. 날짜
2. 모임
3. 경기 추가
4. 경기 카드 리스트
5. 메모
6. 저장 버튼

### UX 규칙

- 날짜는 오늘로 자동 설정한다.
- 최근 모임을 기본 선택한다.
- 최근 사람을 먼저 보여준다.
- 경기 추가 버튼은 항상 눈에 띄게 둔다.

---

## 8.3 경기 추가 화면

### 목적

단식/복식 경기 결과 입력.

### 구성

1. 단식 / 복식 선택
2. 내 팀 선택
3. 상대 팀 선택
4. 점수 입력
5. 승패 자동 계산
6. 저장

### 입력 단순화

- 복식 선택 시 파트너 1명, 상대 2명 선택 UI를 보여준다.
- 단식 선택 시 상대 1명만 선택한다.
- 점수는 숫자 키패드를 사용한다.

---

## 8.4 통계 화면

### 목적

사람별, 모임별, 대회별 승률 확인.

### 탭

- 전체
- 사람별
- 모임별
- 대회별

### 구성

- 기간 필터
- 요약 카드
- 리스트
- 상세 진입

### 통계 표현 규칙

- 승률만 보여주지 말고 경기 수도 함께 표시한다.
- 경기 수가 적은 경우 “기록이 더 필요해요” 안내를 보여준다.
- 데이터가 충분하지 않은 승률은 과도하게 강조하지 않는다.

## 9. 네비게이션

## 9.1 하단 탭

```text
홈 | 기록 | 통계 | 관리 | 내 정보
```

## 9.2 탭별 역할

| 탭 | 역할 |
|---|---|
| 홈 | 핵심 지표와 빠른 기록 |
| 기록 | 운동일지 목록과 작성 |
| 통계 | 승률 분석 |
| 관리 | 사람, 모임, 대회 관리 |
| 내 정보 | 급수와 설정 |

## 9.3 플로팅 액션 버튼

- 홈과 기록 화면에서 `+ 기록하기` 버튼을 제공한다.
- 버튼은 하단 탭과 겹치지 않도록 safe area를 고려한다.

- 승률 링 차트
- 최근 경기 결과 미니 바
- 모임별 승률 막대 그래프
- 기간별 승률 라인 차트

## 10.2 차트 규칙

- 차트에는 반드시 숫자 라벨을 함께 표시한다.
- 빨강/초록만으로 승패를 구분하지 않는다.
- 데이터가 적을 때는 차트를 과하게 보여주지 않는다.

## 10.3 데이터 없음 상태

```text
아직 승률을 계산할 경기 기록이 부족해요.
3경기 이상 기록하면 더 의미 있는 통계를 볼 수 있어요.
```

## 11. 인터랙션 가이드

## 11.1 기록 저장 피드백

- 저장 성공: 짧은 체크 애니메이션 + “기록했어요!”
- 저장 실패: 명확한 에러 메시지 + 다시 시도 버튼

## 11.2 승패 입력

- 승리 선택 시 긍정적 강조
- 패배 선택 시 과한 부정 표현 금지
- 점수 입력 시 자동 승패 계산

## 11.3 통계 상세 진입

- 카드 탭 시 상세 화면으로 이동
- 상세 화면 진입 시 숫자 카운트업은 1회만 실행

## 12. 접근성 가이드

## 12.1 터치 영역

- 주요 버튼은 44px 이상의 높이를 권장한다.
- 작은 아이콘 버튼도 터치 가능한 영역을 충분히 확보한다.
- 최소 기준은 WCAG 2.2 Target Size 기준을 충족하도록 설계한다.

## 12.2 스크린 리더

React Native 컴포넌트에 다음 속성을 적용한다.

```tsx
<TouchableOpacity
  accessible={true}
  accessibilityRole="button"
  accessibilityLabel="경기 기록 추가"
>
  <Text>+ 경기 추가</Text>
</TouchableOpacity>
```

## 12.3 색상 대비

- 본문 텍스트는 배경과 충분한 대비를 가진다.
- 투명 카드 위 텍스트는 반드시 대비를 검증한다.
- 승/패 정보는 색상 + 텍스트를 함께 제공한다.

## 12.4 모션 접근성

- 과한 반복 애니메이션을 피한다.
- 사용자가 모션 감소 설정을 사용하면 애니메이션을 줄인다.

## 13. 콘텐츠 가이드

## 13.1 문체

- 짧고 명확하게 쓴다.
- 사용자를 격려한다.
- 운동 기록 앱답게 행동 중심 문구를 사용한다.

## 13.2 버튼 문구

| 상황 | 문구 |
|---|---|
| 운동일지 작성 | 기록하기 |
| 경기 추가 | 경기 추가 |
| 저장 | 저장하기 |
| 수정 | 수정하기 |
| 삭제 | 삭제하기 |
| 통계 보기 | 승률 보기 |

## 13.3 상태 문구

| 상태 | 문구 |
|---|---|
| 기록 없음 | 아직 기록이 없어요. |
| 사람 없음 | 함께 경기한 사람을 추가해보세요. |
| 모임 없음 | 자주 가는 모임을 등록해보세요. |
| 대회 없음 | 참가한 대회를 기록해보세요. |
| 저장 성공 | 기록했어요! |
| 저장 실패 | 저장하지 못했어요. 다시 시도해주세요. |

## 16. MVP 디자인 체크리스트

- [ ] 홈에서 현재 급수와 전체 승률이 보인다.
- [ ] 기록하기 버튼이 명확하다.
- [ ] 경기 1개를 빠르게 추가할 수 있다.
- [ ] 사람 선택이 복잡하지 않다.
- [ ] 모임 선택이 빠르다.
- [ ] 승률 숫자가 명확하다.
- [ ] 승/패가 색상만으로 구분되지 않는다.
- [ ] 기록이 없을 때 빈 상태 문구가 있다.
- [ ] 모든 주요 버튼에 충분한 터치 영역이 있다.
