# PRD: 소중한 나의 배드민턴 일지, “소나배”

- 문서 버전: v1.0
- 작성일: 2026-05-11
- 제품 유형: 모바일 앱
- 기술 스택: React Native, TypeScript, Firebase, Python FastAPI
- 현재 범위: MVP 기획 문서

---

## 1. 제품 개요

### 1.1 제품명

**소나배**  
뜻: **소중한 나의 배드민턴 일지**

### 1.2 한 줄 소개

배드민턴 동호인이 모임, 경기, 사람, 대회, 급수 기록을 남기고 승률과 활동 흐름을 확인하여 실력 향상과 승급을 돕는 개인 운동 기록 앱.

### 1.3 제품 배경

배드민턴 동호인은 정기 모임, 번개 모임, 대회, 파트너 경기, 상대 경기 등 다양한 활동을 한다. 하지만 대부분의 기록은 기억, 단톡방, 사진, 수첩에 흩어져 있어 다음과 같은 문제가 생긴다.

- 내가 누구와 경기했는지 기억하기 어렵다.
- 특정 파트너와의 승률을 알기 어렵다.
- 특정 상대에게 강한지 약한지 파악하기 어렵다.
- 모임별 활동량과 승률을 비교하기 어렵다.
- 대회별 성과를 누적 관리하기 어렵다.
- 현재 급수와 승급 목표를 체계적으로 관리하기 어렵다.

**소나배**는 이 문제를 해결하기 위해 배드민턴 활동을 빠르게 기록하고, 기록된 데이터를 기반으로 승률과 성장 흐름을 보여주는 앱이다.

---

## 2. 문제 정의

### 2.1 핵심 문제

배드민턴 동호인이 모임과 경기 후 자신의 운동 기록, 사람별 승률, 모임별 승률, 대회 성과, 현재 급수를 체계적으로 모니터링하기 어렵다.

### 2.2 사용자 니즈

사용자는 다음을 원한다.

- 운동 후 빠르게 일지를 기록하고 싶다.
- 오늘 누구와 경기했는지 남기고 싶다.
- 사람별 승률을 보고 싶다.
- 모임별 활동 기록과 승률을 보고 싶다.
- 대회마다 승률과 결과를 확인하고 싶다.
- 현재 급수와 승급 목표를 관리하고 싶다.
- 복잡한 로그인 없이 바로 기록하고 싶다.

## 3. 목표와 비목표

### 3.1 제품 목표

1. 배드민턴 운동일지를 쉽게 기록한다.
2. 사람별, 모임별, 대회별 승률을 자동 계산한다.
3. 사용자의 현재 급수와 승급 목표를 기록하고 추적한다.
4. 사용자가 자신의 운동 패턴과 성과를 한눈에 확인할 수 있게 한다.
5. 로그인 없이도 앱을 사용할 수 있는 간단한 진입 경험을 제공한다.

### 3.2 MVP 목표

MVP에서는 다음 기능을 반드시 제공한다.

- 운동일지 작성
- 사람별 승률 확인
- 모임별 승률 확인
- 모임 기록 관리
- 대회별 승률 확인
- 현재 나의 급수 기록 및 확인

### 3.3 비목표

MVP에서는 다음을 우선순위에서 제외한다.

- 소셜 로그인
- 공개 커뮤니티
- 채팅
- 결제
- 광고
- 랭킹 경쟁 시스템
- 대회 접수 시스템
- 실시간 경기 중계
- 코치 매칭
- 모임원 초대 기반 협업 기록

## 4. 대상 사용자

### 4.1 주요 사용자

**배드민턴 동호인**

### 4.2 사용자 페르소나

#### Persona A. 꾸준히 모임에 나가는 동호인

- 주 2~4회 배드민턴 모임 참여
- 여러 파트너와 복식 경기 진행
- 자신의 승률과 실력 변화를 알고 싶음
- 기록은 하고 싶지만 복잡한 입력은 싫어함

#### Persona B. 승급을 목표로 하는 동호인

- 현재 급수에서 다음 급수로 올라가고 싶음
- 대회 성적과 모임 경기 결과를 관리하고 싶음
- 어떤 상대, 어떤 파트너와 성과가 좋은지 알고 싶음

#### Persona C. 여러 모임을 다니는 동호인

- A모임, B모임, 지역 모임 등 여러 곳에서 운동
- 모임마다 경기 수준과 승률 차이를 비교하고 싶음
- 활동량을 누적해서 보고 싶음

## 5. 핵심 사용자 시나리오

### 5.1 운동 후 일지 기록

1. 사용자가 모임 운동을 마친다.
2. 앱을 실행한다.
3. “기록하기”를 누른다.
4. 날짜, 모임, 경기 방식, 함께 경기한 사람, 점수, 승패를 입력한다.
5. 앱이 자동으로 사람별, 모임별 승률을 갱신한다.

### 5.2 사람별 승률 확인

1. 사용자가 통계 화면으로 이동한다.
2. “사람별” 탭을 선택한다.
3. 특정 사람과 파트너였을 때의 승률, 상대였을 때의 승률, 함께 경기한 횟수를 확인한다.

### 5.3 모임별 승률 확인

1. 사용자가 통계 화면에서 “모임별” 탭을 선택한다.
2. 모임 리스트를 확인한다.
3. 각 모임의 경기 수, 승, 패, 승률, 최근 활동일을 확인한다.

### 5.4 대회별 성과 확인

1. 사용자가 대회 기록을 추가한다.
2. 대회명, 날짜, 급수, 경기 결과를 기록한다.
3. 앱이 대회별 승률과 누적 성과를 보여준다.

### 5.5 현재 급수와 승급 목표 확인

1. 사용자가 내 정보 화면에서 현재 급수를 설정한다.
2. 목표 급수를 설정한다.
3. 홈 화면에서 현재 급수, 목표 급수, 최근 승률, 대회 성과를 확인한다.

## 7. 기능 상세 요구사항

## FR-001. 초기 프로필 설정

### 설명

사용자는 앱 첫 실행 시 로그인 없이 간단한 프로필을 설정한다.

### 입력 항목

- 닉네임
- 현재 급수
- 목표 급수

### 급수 예시

급수 체계는 지역, 동호회, 대회마다 다를 수 있으므로 고정값과 사용자 정의를 모두 지원한다.

기본 예시:

- 초심
- D조
- C조
- B조
- A조
- S조
- 사용자 정의

### 정책

- 로그인 화면은 제공하지 않는다.
- 내부적으로는 데이터 저장을 위해 익명 사용자 ID를 생성한다.
- 사용자는 “계정 생성”이라는 부담 없이 바로 기록을 시작한다.

### 완료 조건

- 사용자는 첫 실행 후 1분 안에 기록 준비 상태가 되어야 한다.
- 닉네임과 급수는 나중에 수정 가능해야 한다.

### 입력 항목

### 기본값

- 날짜: 오늘

- 경기 방식: 최근 사용한 경기 방식

## FR-007. 사람별 승률

### 설명

사용자는 특정 사람과 관련된 승률을 확인한다.

### 통계 항목

- 총 경기 수
- 승
- 패
- 승률
- 최근 경기일
- 함께 파트너였을 때 승률
- 상대였을 때 나의 승률
- 해당 사람과 가장 많이 경기한 모임

### 계산 기준

```text
전체 승률 = 승리 수 / 전체 경기 수 * 100
파트너 승률 = 해당 사람과 같은 팀이었던 경기의 승리 수 / 같은 팀 경기 수 * 100
상대 승률 = 해당 사람이 상대 팀이었던 경기의 승리 수 / 상대 경기 수 * 100
```

### 완료 조건

- 사람별 승률 리스트를 볼 수 있어야 한다.
- 사람 상세 화면에서 파트너 기준과 상대 기준 승률을 구분해서 볼 수 있어야 한다.

## FR-008. 모임별 승률

### 설명

사용자는 모임별 활동량과 승률을 확인한다.

### 통계 항목

- 모임명
- 총 운동일 수
- 총 경기 수
- 승
- 패
- 승률
- 최근 활동일
- 가장 많이 함께한 사람

### 계산 기준

```text
모임 승률 = 해당 모임에서 승리한 경기 수 / 해당 모임 전체 경기 수 * 100
```

### 완료 조건

- 모임별 승률 리스트를 볼 수 있어야 한다.
- 모임 상세 화면에서 최근 운동일지와 경기 기록을 볼 수 있어야 한다.

## 11. 통계 계산 정의

## 11.1 기본 승률

```text
승률 = 승리 수 / (승리 수 + 패배 수) * 100
```

`UNKNOWN` 결과는 승률 계산에서 제외한다.

## 11.2 전체 승률

```text
전체 승률 = 전체 승리 수 / 전체 결과 확정 경기 수 * 100
```

## 11.3 사람별 승률

### 파트너 승률

```text
파트너 승률 = 특정 사람과 같은 팀으로 경기한 승리 수 / 특정 사람과 같은 팀으로 경기한 전체 수 * 100
```

### 상대 승률

```text
상대 승률 = 특정 사람이 상대 팀에 있었던 경기의 승리 수 / 특정 사람이 상대 팀에 있었던 전체 경기 수 * 100
```

## 11.4 모임별 승률

```text
모임 승률 = 특정 모임에서의 승리 수 / 특정 모임에서의 결과 확정 경기 수 * 100
```

## 11.5 대회별 승률

```text
대회 승률 = 특정 대회에서의 승리 수 / 특정 대회에서의 결과 확정 경기 수 * 100
```

## 12. 기술 아키텍처 초안

## 12.1 전체 구조

```text
React Native Mobile App
        ↓
Firebase Anonymous Auth 또는 내부 익명 ID
        ↓
Cloud Firestore
        ↓
Python FastAPI 통계/분석 API, 선택 적용
```

## 12.2 기술 역할

| 기술 | 역할 |
|---|---|
| React Native | iOS/Android 모바일 앱 개발 |
| TypeScript | 타입 안정성 확보 |
| Firebase Firestore | 운동일지, 사람, 모임, 경기 기록 저장 |
| Firebase Auth Anonymous | 로그인 없는 사용자 식별, 선택 적용 |
| Python FastAPI | 통계 계산, 리포트, 향후 AI 분석 API |

## 12.3 FastAPI 사용 방향

MVP에서는 앱 내부 또는 Firebase 쿼리 기반으로 통계를 계산해도 된다. FastAPI는 다음 기능이 필요해질 때 도입한다.

- 복잡한 승률 집계
- 기간별 분석 리포트
- 승급 가능성 분석
- 추천 파트너 분석
- 데이터 백업/내보내기 처리

### API 예시

```http
GET /stats/summary?userId={userId}
GET /stats/person/{personId}?userId={userId}
GET /stats/meeting/{meetingId}?userId={userId}
GET /stats/tournament/{tournamentId}?userId={userId}
POST /stats/recalculate
```

## 13. 비기능 요구사항

## 13.1 사용성

- 운동 후 빠르게 기록할 수 있어야 한다.
- 경기 1개 입력은 30초 이내 완료를 목표로 한다.
- 최근 선택한 모임과 사람을 우선 노출한다.

## 13.2 성능

- 홈 화면 로딩은 일반적인 네트워크 환경에서 2초 이내를 목표로 한다.
- 통계 화면은 1,000개 경기 기록 기준으로도 자연스럽게 동작해야 한다.

## 13.3 접근성

- 버튼과 입력 요소는 충분한 터치 영역을 제공한다.
- 통계는 색상만으로 구분하지 않고 텍스트/숫자도 함께 제공한다.
- React Native 접근성 속성을 적용한다.

## 13.4 보안 및 개인정보

- 로그인은 없지만 내부 사용자 식별자는 분리 관리한다.
- 다른 사용자의 기록에 접근할 수 없도록 Firestore 보안 규칙을 설정한다.
- 개인 이름은 사용자가 직접 입력하므로 민감정보로 취급한다.

## 13.5 데이터 안정성

- 앱 종료 후 기록이 유지되어야 한다.
- 네트워크 오류 시 저장 실패를 사용자에게 알려야 한다.
- 가능하면 임시 저장 또는 재시도 처리를 제공한다.

## 19. 최종 MVP 정의

**소나배 MVP는 배드민턴 동호인이 로그인 없이 앱을 시작하여 운동일지, 사람, 모임, 대회, 급수를 기록하고 사람별/모임별/대회별 승률을 확인할 수 있는 모바일 앱이다.**

가장 중요한 제품 가치는 다음 세 가지다.

1. 기록이 쉬워야 한다.
2. 승률이 정확해야 한다.
3. 승급을 위한 나의 상태가 한눈에 보여야 한다.
